🕷️ CrawlDoctor - AI Crawler Tracking

Comprehensive tracking system for AI crawlers like ChatGPT, Claude, Perplexity, and Google AI Studio

📊 System Overview

CrawlDoctor provides multiple tracking methods to capture AI crawler visits that traditional analytics miss:

🖼️ Pixel Tracking

1x1 transparent pixel that works with all crawlers, even those without JavaScript support.

📡 Beacon API

Advanced tracking with structured data and custom events for detailed analytics.

📁 Resource Tracking

Alternative tracking through CSS, JS, or JSON resource requests.

🔍 AI Detection

Sophisticated crawler detection using user agent patterns and behavioral analysis.

🚀 Quick Start - Pixel Tracking

Method 1: Simple Image Pixel

The most compatible method - works with all crawlers including those that don't execute JavaScript:

<!-- Basic pixel tracking --> <img src="https://your-domain.com/track/pixel.gif" alt="" width="1" height="1" style="display:none;" /> <!-- With page URL tracking --> <img src="https://your-domain.com/track/pixel.gif?url=https://yoursite.com/page" alt="" width="1" height="1" style="display:none;" /> <!-- With tracking ID and campaign --> <img src="https://your-domain.com/track/pixel.gif?url=https://yoursite.com/page&tid=blog-2024&campaign=content-marketing" alt="" width="1" height="1" style="display:none;" />
💡 Pro Tip: Place this pixel at the bottom of your page content for best results.

Method 2: JavaScript Enhanced Tracking

Provides fallback compatibility and additional features:

<!-- JavaScript with pixel fallback --> <script> (function() { var img = new Image(); img.src = 'https://your-domain.com/track/pixel.gif?' + 'url=' + encodeURIComponent(window.location.href) + '&title=' + encodeURIComponent(document.title) + '&tid=your-tracking-id'; })(); </script> <!-- Fallback pixel for non-JS crawlers --> <noscript> <img src="https://your-domain.com/track/pixel.gif?url=https://yoursite.com/page" alt="" width="1" height="1" style="display:none;" /> </noscript>

⚡ Advanced Tracking

Method 3: Full JavaScript SDK

Complete tracking solution with event tracking and real-time analytics:

<!-- Load CrawlDoctor SDK --> <script src="https://your-domain.com/track/embed.js?tracking_id=your-id&auto_track=true"></script> <!-- Manual page tracking --> <script> CrawlDoctor.trackPageView(window.location.href, document.title); </script> <!-- Custom event tracking --> <script> CrawlDoctor.trackEvent('engagement', 'scroll_50_percent', { page: window.location.pathname, timestamp: new Date().toISOString() }); </script>

Method 4: Server-Side Integration

For dynamic content and server-side rendering:

<!-- PHP Example --> <?php $tracking_url = 'https://your-domain.com/track/pixel.gif?' . http_build_query([ 'url' => $_SERVER['REQUEST_URI'], 'title' => $page_title, 'tid' => 'server-side-tracking', 'source' => 'organic' ]); ?> <img src="<?= htmlspecialchars($tracking_url) ?>" alt="" width="1" height="1" style="display:none;" /> <!-- Node.js/Express Example --> const trackingUrl = `https://your-domain.com/track/pixel.gif?${new URLSearchParams({ url: req.originalUrl, title: pageTitle, tid: 'node-tracking', source: 'server' })}`;

⚠️ Implementation Notes

🔒 Privacy: This system respects privacy by only tracking crawlers, not human visitors. Configure your privacy policy accordingly.
📈 Performance: Pixel tracking has minimal performance impact - the 1x1 pixel is only 43 bytes and loads asynchronously.
🌍 GDPR: While crawler tracking is generally acceptable, ensure compliance with local privacy regulations.

📋 Testing Your Implementation

Verify your tracking is working correctly:

  1. Check Network Tab: Look for requests to /track/pixel.gif in browser dev tools
  2. Monitor Dashboard: Log into your CrawlDoctor dashboard to see real-time data
  3. Test Different User Agents: Use browser dev tools to simulate different crawlers
  4. Verify Pixel Load: Ensure the tracking pixel loads without errors
# Test with curl to simulate a crawler curl -H "User-Agent: GPTBot/1.0" "https://your-domain.com/track/pixel.gif?url=test" # Check if request was tracked curl -H "Authorization: Bearer YOUR_TOKEN" "https://your-domain.com/api/v1/analytics/realtime"

🛠️ API Integration

For programmatic access to your tracking data:

// Get analytics summary fetch('https://your-domain.com/api/v1/analytics/summary', { headers: { 'Authorization': 'Bearer YOUR_TOKEN' } }) .then(response => response.json()) .then(data => console.log(data)); // Export data fetch('https://your-domain.com/api/v1/analytics/export?format=csv', { headers: { 'Authorization': 'Bearer YOUR_TOKEN' } }) .then(response => response.blob()) .then(blob => { const url = URL.createObjectURL(blob); const a = document.createElement('a'); a.href = url; a.download = 'crawldoctor-data.csv'; a.click(); });
